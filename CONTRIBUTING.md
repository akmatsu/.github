# Contributing Guidelines

Thank you for helping improve our projects!

## Discuss First

Before starting a major change or new feature, open an issue to confirm alignment and avoid duplicate work.

## Code Style

Follow the project's coding standards and use any provided linting/formatting tools. Check the project's [README](/README.md) for more details.

##  Commit Messages

Use clear, descriptive commit messages. Conventional commits are preferred. (E.G, `feat: add new feature`, `fix: resolved bug`).

## Pull Requests

When submitting a pull request:

- Keep titles concise and descriptive.
- Reference related issues, if any.
- Ensure tests pass before submitting.

## Reporting Issues

- Please open an issue.

---

We appreciate your interest in contributing!
